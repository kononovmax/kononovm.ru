Дизайн-система Istock.info

Проблема

Istock.info - это сервис, который позволяет работать с товарами. В нем десятки тысяч разных позиций, которые пользователи создают и по цепочке отправляют друг другу. Для представления этих данных и управления ими используются интерфейсы с десятками видов таблиц и навигации. При проектировании такого сервиса количество макетов и компонентов исчисляется тысячами. Без продуманной Дизайн-системы время, которое тратится на создание и поддержку макетов, увеличивается в десятки раз. Создаются разные по виду компоненты, которые выполняют одну функцию, что вводит в заблуждение пользователей и добавляет проблем разработке.

Задачи

Улучшение читаемости макетов Уменьшение набора компонентов
Ускорение проектирования Улучшение внешнего вида сервиса
Упрощение поддержки Дизайн-системы

Сценарии

Это самая большая часть Дизайн-системы. Каждый сценарий показывает один этап взаимодействия пользователя с сайтом. Сценарий состоит из функциональной схемы, которая показывает действия пользователя и системы, а также последовательности шаблонов (о них рассказано ниже), которые демонстрируют интерфейс. Для упрощения ориентирования в макетах шаблоны и схемы пронумированы и подписаны.

Шаблоны

Это готовые экраны с разными видами интерфейса. Они используются для ускорения проектирования. Шаблоны состоят из Блоков и Компонентов (о них рассказано ниже). В шаблонах есть возможность динамически менять ширину, что позволяет тестировать корректное отображение на разных девайсах.

Блоки

Это вспомогательные элементы, в которых можно менять количество контента. Блоки отлично подходят для проектирования Таблиц и Сайдбаров благодаря возможности быстро добавлять и удалять строки в них. Внутри блоков могут содержаться Компоненты или другие Блоки.

Компоненты

Компонент создается один раз. В Шаблонах и Блоках используются его копии, которые обновляются при каждом изменении оригинала. Компоненты можно группировать в Вариации. Это позволяет быстро переключаться между различающимися модификациями одного Компонента. Компонент может содержать в себе другие Компоненты.
